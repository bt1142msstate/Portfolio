# Portfolio Roadmap

This roadmap tracks the planned direction for BrandonTemple.com. Version 1 stays focused on hiring and software credibility. Later versions can expand the site into a broader personal brand and creator presence.

## Version 1: Software Portfolio Foundation

Status: Work in progress.

Primary goal: make the site strong for recruiters, hiring managers, and people evaluating software engineering work.

Planned before v1 is considered complete:

- Add one more featured project for the mobile app once it is ready.
- Present the app as a polished iOS and Android project, not just another code repository.
- Add app links when available, such as App Store, Google Play, landing page, or demo.
- Decide the publishing and business setup for the app, including whether to release it under an LLC or another app publisher identity.
- Clarify the app monetization model at a high level: free core app with subscription-based extra features.
- Move portfolio contact to a domain-based email address once it is set up.
- Update public profile links if the GitHub handle changes to a more name-based account.
- Keep the public site recruiter-focused, clean, and professional.

What should not be added to v1:

- High school education.
- Long personal bio content.
- Influencer-focused sections.
- Photos or lifestyle content beyond professional brand assets.

## Version 2: Personal Brand And Creator Expansion

Primary goal: make the site feel more alive while keeping the software portfolio credible.

Likely direction:

- Add photos and stronger personal visuals.
- Add a fuller bio that covers Brandon as a person, not only as a software developer.
- Add content creation and influencer presence in a dedicated area.
- Add social links and creator-platform context where appropriate.
- Consider a separate page such as `/about/`, `/creator/`, or `/media/`.
- Add a subtle main-site path to the expanded profile, such as a `More About Me` button.
- Keep resume and professional experience focused on software work.

Open design question:

- Decide whether personal and creator content belongs directly on the homepage or on a separate page so the homepage can stay software-first.

## Version 3: Expanded Personal Platform

Primary goal: build on v2 once the creator direction and audience are clearer.

Possible direction:

- Expand the site into a richer personal platform.
- Improve how search engines and AI systems understand Brandon's work, background, content, and public profiles.
- Add more structured content if there is enough public material to support it.
- Revisit navigation, content organization, and search visibility after v2 is live.

Version 3 is intentionally not tightly scoped yet. The right direction should come from what works in v1 and v2.
